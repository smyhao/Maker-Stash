# 后端第一阶段决策

## 范围

第一阶段先做后端，不做前端。P0 聚焦可用闭环：

- 物品、分类、位置
- 搜索
- 备注 / 操作记录
- 图片与附件元信息
- 备份
- API Token
- 常用与补货字段供前端展示
- 最小 CLI 客户端，覆盖配置、健康检查、物品列表/查看/新增、移动、数量动作、补货/常用、标签、别名、外部标识、位置、搜索、备份
- 属性模板和物品属性 API，支撑前端高级字段编辑器
- 附件下载和备份下载

标签、别名、外部标识、标签打印、NFC、AI 编排作为后续阶段，但模型和接口位置先预留。

## 技术选择

- FastAPI
- SQLAlchemy 2.x
- Pydantic
- SQLite
- Alembic 管理数据库结构迁移

## 关键规则

- 物品编号按分类前缀递增，例如 `FIL-000001`。
- 位置 `code/full_code` 第一版创建后不允许修改。
- Token 只存哈希，初始化脚本只显示一次明文 Token。
- 删除物品默认归档；如用户二次确认删除附件，则附件元信息标记删除。
- 备份恢复前必须先创建当前快照。
- CLI 只调用 API，配置保存在用户目录 `~/.workshop-stash/config.toml`。
- 数据库结构通过 Alembic 迁移固化，默认数据通过 `app.scripts.init_db` 初始化。
- 分类和位置树接口返回嵌套 `children`，前端不需要自行从平铺列表组树。
- 物品列表 API 支持 `page/page_size`，`page_size` 限制为 1 到 100。
- FastAPI 参数校验错误统一返回 `VALIDATION_ERROR`。
