# 扩展开发文档

这里集中放置未来开发 Maker Stash 扩展时需要遵守和参考的文档。扩展指扫码、标签打印、批量导入、Agent 自动化、外部脚本等通过主干 API 增加能力的独立模块。

## 阅读顺序

1. [扩展开发 Rule](RULE.md)  
   先读。定义扩展开发边界、禁止事项、写操作规则和验收要求。

2. [快速开始](QUICKSTART.md)  
   给新扩展建立最小调用闭环：连接主干、读取 capabilities、发起查询和安全写入。

3. [API 调用规范](API.md)  
   作为长期契约参考：认证、幂等、来源字段、能力发现、错误码和版本策略。

4. [Workflow 使用指南](WORKFLOW.md)  
   用于批量导入、批量出库、Agent 操作等高风险写入场景。

## 总原则

- 扩展优先通过 REST API 访问主干，CLI 只作为人工操作、调试和简单脚本入口。
- 扩展不要直连 SQLite，不要绕过 API 修改上传目录，不要依赖前端内部类型。
- 扩展写操作必须可追踪、可重试、可审计。
- 批量、库存扣减、位置整理和 Agent 自动写入优先走 workflow。
- 扩展需要主干新增能力时，先把它设计成通用 API 契约，再补 capabilities、文档和测试。
