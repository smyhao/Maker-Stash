# 工坊物栈 v3 Prompt：连续快速录入模式

## 1. 你的角色

你是本项目的高级前端工程师兼产品工程师。  
你的任务是在当前 Maker Stash / 工坊物栈代码基础上，设计并实现一个“连续快速录入模式”，用于用户边整理真实物品边快速建立物品档案。

请先阅读项目现有结构、`README.md`、`AGENTS.md`，以及相关前端/后端代码，再动手实现。  
不要脱离现有界面重做一套新设计；必须延续当前工坊物栈的布局、配色、组件密度和交互风格。

---

## 2. 产品背景

当前项目定位是：

```text
个人工坊物品管理系统
档案优先，库存能力保留
当前阶段主要录入电子元器件，其次是工具、螺丝、耗材等
用户主要场景是：买回来实物后，站在收纳盒/抽屉前边整理边录入
```

当前已有能力：

```text
物品 CRUD
分类树
位置树与收纳盒/格位
数量、单位、状态、补货、常用
标签、属性、附件、封面图
后端已有属性模板 attribute_definitions
前端已有普通新增物品弹窗 ItemFormDialog
```

但普通新增弹窗更适合单条录入，不适合连续整理几十个元器件。

本功能要解决的问题：

```text
用户整理同一个抽屉/盒子时，可以锁定默认分类和主位置，连续录入多个物品。
每条物品只录最关键字段，保存后快速进入下一条。
系统保留本次录入列表，方便立即回看和修正。
```

## 6. 本次整理设置

提供一个设置卡片：

```text
本次整理设置
```

字段：

```text
默认分类
主位置
锁定分类
锁定位置
```

行为：

```text
默认分类用于新建物品的 category。
主位置优先使用 location_code；如果未选择结构化位置，可允许 location_text。
锁定分类开启时，保存后下一条继续使用相同分类。
锁定位置开启时，保存后下一条继续使用相同位置。
```

建议默认：

```text
锁定分类：开启
锁定位置：开启
```

原因：

```text
连续整理同一个抽屉/盒子时，分类和位置通常连续重复。
```

## 7. 快速添加表单

表单标题：

```text
快速添加物品
```

首版字段：

```text
名称：必填
分类：默认来自本次整理设置，可临时覆盖
主位置：默认来自本次整理设置，可临时覆盖
数量：可空
单位：可空
状态：默认 normal / 正常
标签：可空，逗号分隔
备注：可空
```

字段原则：

```text
名称用于快速识别和搜索。
分类用于粗粒度导航。
属性用于结构化规格，不要强迫首次录入完整填写。
数量按用户输入精度展示，不要补成 100.00。
```

电子元器件示例：

```text
名称：10k 电阻
分类：元器件 / 电阻
主位置：箱子2 · A03
数量：100
单位：个
标签：0603, 1%
备注：可选
```

## 8. 快速属性

首版可以实现为轻量选填区域：

```text
快速属性
```

要求：

```text
如果能低成本复用后端 attribute_definitions，则选择分类后展示该分类启用的属性模板。
用户可以填写其中一部分，不必全部填写。
保存物品时，将已填写属性写入 item attributes。
```

如果接入属性模板成本过高，可以首版先做成后续 TODO，但代码结构不要阻碍后续接入。

推荐属性展示方式：

```text
小型输入行或紧凑 chip，不要做复杂大表单。
```

示例：

```text
阻值 = 10k
封装 = 0603
精度 = 1%
功率 = 1/10W
```

注意：

```text
属性模板是辅助，不是必填门槛。
```

## 9. 表单动作

底部动作：

```text
复制上一条
保存并查看详情
保存并录入下一个
```

行为定义：

### 保存并录入下一个

```text
调用现有 createItem API 创建物品。
成功后将物品加入“本次已录入”列表。
清空名称、数量、标签、备注、快速属性值。
根据锁定状态保留或清空分类/位置。
焦点回到名称输入框。
```

### 保存并查看详情

```text
创建物品后跳转到库存页，选中新建物品并打开详情。
```

### 复制上一条

```text
将上一条录入物品的分类、位置、单位、状态、标签、属性复制到当前表单。
名称可以复制，但建议选中文本，方便用户直接修改。
数量是否复制可按实现判断；如果复制，需要方便用户快速改值。
```

## 17. 最终判断

这个功能的核心不是增加一个更大的表单，而是把真实整理场景中的重复动作压缩掉：

```text
分类少选一次，位置少选一次，保存后少点一次，上一条少填一次。
```

首版只要把这四件事做顺，就比继续堆字段更有价值。
