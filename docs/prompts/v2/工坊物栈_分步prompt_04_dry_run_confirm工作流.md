# 工坊物栈分步 Prompt 04：dry-run / confirm 工作流

## 1. 前提

只有在前 3 步完成后，才执行这一步。  
如果当前还没有稳定底层、幂等、审计和任务 API，这一步不要抢跑。

---

## 2. 目标

本次只做：

```text
为批量导入、批量出库和 Agent 操作补上 plan / confirm 工作流
```

核心原则：

```text
先计划
再确认
最后执行
```

---

## 3. 必做内容

### 3.1 plan / dry-run

至少支持这些场景：

```text
批量导入预演
批量出库预演
后续 Agent 执行前计划输出
```

plan 结果建议至少包含：

```text
将创建哪些记录
将更新哪些记录
将跳过哪些记录
哪些记录会失败
风险提示
确认执行所需标识
```

### 3.2 confirm 执行

要求：

```text
确认执行必须基于 plan 结果继续
不得在 confirm 时重新自由发挥
要兼容幂等和审计
必要时可走任务系统
```

### 3.3 工作流契约

统一这类接口的调用规范：

```text
plan 请求结构
plan 响应结构
confirm 请求结构
confirm 响应结构
与 job 的衔接方式
```

---

## 4. 设计边界

本阶段不要做：

```text
复杂规则引擎
通用可视化编排器
独立中间层服务拆分
```

如果需要工作流层，请优先作为同一后端中的逻辑层实现，而不是提前拆服务。

---

## 5. 测试要求

至少补这些测试：

```text
plan 只预览不落库
confirm 才真正执行
重复 confirm 不会重复副作用
plan / confirm 结构稳定
与任务和审计可正常协同
```

---

